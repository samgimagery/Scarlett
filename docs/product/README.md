# Scarlett Product Docs

Scarlett is a managed customer-service system built from a business's real knowledge.

Product split:

- **Scarlett Core** — reusable engine, ingestion, RAG, service flow, facts, channels, review tools.
- **Customer Instance** — business-specific vault, sources, rules, channels, logs, tests, deployment config.

The sales model is done-for-you: prepare remotely, demo live, launch and tune as a managed system.

- [Scarlett Core Status — 2026-05-07](Scarlett-Core-Status-2026-05-07.md) — live AMS hardening checkpoint: router, pricing, continuing-ed, handoff, service tiles, voice assets, gates, and next priority.
- [Scarlett Live Voice Checkpoint — 2026-05-07](Scarlett-Live-Voice-Checkpoint-2026-05-07.md) — cached-first browser voice state: approved AMS WAV assets, live playback wiring, real-device feedback, contextual starter bank, gates, and REQ-161 listening plan.

Voice is part of Scarlett Core, not a one-off demo. The locked live-voice architecture is cached-first: use situation-specific service tiles for immediate first audio, use contextual starters only when lookup or answer preparation is actually slow, then deliver short grounded answer chunks with composed pacing.
